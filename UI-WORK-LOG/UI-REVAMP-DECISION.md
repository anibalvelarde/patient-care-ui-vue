# UI Revamp — Decision Point

After evaluating 7 design options side-by-side (plus the original UI), **Option 01: Clean Medical** was selected as the primary UI for the application.

---

## Decision

- **Chosen design**: Option 01 — Clean Medical
- **Rationale**: Professional blue/teal palette, top navigation bar, card-based AM/PM appointment tables with status pills. Clean, modern layout appropriate for a therapy center front-desk workflow.
- **Other options**: Kept in the codebase for reference. Design metaphors from other options may be borrowed in the future.

---

### 3. API Base URL Precedence Fix

Fixed `HttpClientBase.getBaseUrlFromConfig()` so the config source priority matches the documented behavior in `public/config.js`:

- **Dev mode** (`npm run dev`): `.env` (`VITE_API_BASE_URL`) overrides the static `public/config.js` placeholder
- **Production** (Docker/K3s): Runtime `config.js` (generated by Docker entrypoint) always wins — `VITE_API_BASE_URL` is ignored even if accidentally baked into the build

This uses Vite's `import.meta.env.DEV` flag, which is dead-code-eliminated in production builds.

```typescript
if (import.meta.env.DEV) {
  return import.meta.env.VITE_API_BASE_URL || appConfig?.apiBaseUrl || "http://localhost:5245";
}
return appConfig?.apiBaseUrl || "http://localhost:5245";
```
