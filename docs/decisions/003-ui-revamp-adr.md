# ADR-003: UI Revamp — Initial Design Selection

## Status

**Superseded.** The original 2026-03-12 selection of "Option 01: Clean Medical" was reversed two days later: on 2026-03-14 (commit `ed65c1b` — "Switch default UI from Option 01 (Clean Medical) to Option 02 (Dashboard Cards)") the team switched the production layout to **Option 02: Dashboard Cards**. That switch was not formally captured as a separate ADR. WP-16C (2026-05-04) finalized this by removing all remaining design-comparison artifacts and making Option 02 the only layout in the codebase.

This document is preserved as the historical record of the original selection process. The decision contents below describe the state as of 2026-03-12 and are no longer current.

## Date

2026-03-12 (original); superseded 2026-03-14; comparison framework dismantled 2026-05-04.

## Context

After evaluating 7 design options side-by-side (plus the original UI), the team needed to pick a primary UI to anchor further development on, rather than continuing to maintain all options indefinitely. See ADR-002 for the comparison framework that produced the candidates.

## Original decision (2026-03-12)

- **Chosen design:** Option 01 — Clean Medical
- **Rationale:** Professional blue/teal palette, top navigation bar, card-based AM/PM appointment tables with status pills. Clean, modern layout appropriate for a therapy center front-desk workflow.
- **Other options:** Kept in the codebase for reference. Design metaphors from other options may be borrowed in the future.

### Changes made (branch `features/claude/ui-style-decision-point`)

#### 2. Navigation Links Updated

#### 3. API Base URL Precedence Fix

Fixed `HttpClientBase.getBaseUrlFromConfig()` so the config source priority matches the documented behavior in `public/config.js`:

- **Dev mode** (`npm run dev`): `.env` (`VITE_API_BASE_URL`) overrides the static `public/config.js` placeholder
- **Production** (Docker/K3s): Runtime `config.js` (generated by Docker entrypoint) always wins — `VITE_API_BASE_URL` is ignored even if accidentally baked into the build

This uses Vite's `import.meta.env.DEV` flag, which is dead-code-eliminated in production builds.

```typescript
if (import.meta.env.DEV) {
  return import.meta.env.VITE_API_BASE_URL || appConfig?.apiBaseUrl || "http://localhost:5245";
}
return appConfig?.apiBaseUrl || "http://localhost:5245";
```

#### 5. What was next (from 2026-03-12 perspective)

> With Clean Medical as the foundation, the next phase focuses on building out real application functionality beyond the daily schedule view — patient management, therapist views, billing, etc.

## What actually happened

- 2026-03-14: switched to Option 02 (Dashboard Cards) — commit `ed65c1b`. The HttpClientBase fix from change #3 above was kept.
- 2026-03-12 onward: real application functionality built out as anticipated (patient management, therapist views, billing, treatment plans, schedules, statements, admin) — but built against the Option 02 layout, not Option 01.
- 2026-05-04 (WP-16C): all design-option views, components, and the comparison framework were removed; `components/option02/` is now the only layout.

## Consequences (current, as of WP-16C)

- The codebase no longer carries the design-comparison framework or rejected-option code.
- `components/option02/` will likely be renamed to `components/layout/` (or similar) under follow-up WP-17C — the `option02` name is a historical artifact, not a meaningful distinction anymore.
- ADR-002 (UI Design Comparison Framework) is also marked Superseded; together these two ADRs document the design-trial era from start to dismantling.
